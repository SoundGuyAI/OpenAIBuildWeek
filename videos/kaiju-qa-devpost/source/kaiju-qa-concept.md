## Pitch

Before a helpful baby kaiju is deployed to a real city, build a tabletop test
district, stage edge cases, and prove that its "help everyone" behavior is safe
enough to ship.

## Player loop

1. Run an empty-street/stalled-car happy path and earn a partial pass.
2. Add a fragile tower; the same helpful behavior knocks it over.
3. Authorize a broad "freeze near buildings" guardrail; the tower passes, but
   an ambulance is blocked.
4. Compare both paths and place a targeted slow-zone boundary.
5. Run all scenarios, stamp release, and promote the kaiju to city guardian.

## Product truths

- The player is the quality engineer, not a monster fighter or programmer.
- Test coverage is a playful miniature city.
- Broad fixes can create visible regressions.
- Outcomes are deterministic animation, not destruction physics.
- Equal semantic actions map to mouse, touch, and XR ray/grab.
- Failure is useful evidence and never humiliates the player.
- The finishing gag is an enormous tester applying a tiny approval seal.
