01 / Edutainment systems

Loop Engineer

Employ a mentor. Master the agent loop.

Employ an NPC mentor who teaches agentic software development across SDLC and SDLR through visible loop engineering.

The first 20 seconds

Patch deliberately follows an underspecified feature request, delivers the wrong cube, and asks what done should look like before the next SDLC and SDLR loop.

The loop

  1. Define the feature and observable result
  2. Plan and delegate bounded specialist jobs
  3. Implement, test, and inspect handoffs
  4. Review evidence, learn, and iterate

Same decision, three inputs

DesktopClick cards, drag to stations, Space to run or pause.

MobileTap-select, tap-place, one-finger orbit, large Run button.

XRRay-select or grab cards, place at waist-high stations, press Run.

Free asset starter kit

Primary packUltimate Modular Sci-Fi PackQuaternius - CC0Free full packWorkflow stations, consoles, bounded lanes, and modular lab dressing.Supporting packUltimate Animated Character PackQuaternius - CC0Free full packMentor and specialist-agent character prototypes.

Still custom: SDLC timeline, SDLR loop, task cards, evidence UI, role-specific gestures, and feedback VFX/audio.

What makes it different

The employee teaches the employer to carry a feature across the software development life cycle while using SDLR, bounded ownership, tests, feedback, and evidence-driven loops instead of programming worker steps.

Hackathon proof

One conveyor scenario, two agent roles, one bad run, and one corrected run.

Comfort

Seated tabletop; no locomotion; captions, undo, and readable evidence.

Risk

Medium originality risk. It may feel like card sorting unless every decision changes visible behavior within seconds.
